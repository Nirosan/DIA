What is DIA.

DIA (Digital Irrigation Automation) is project started with Yarl it Hub with a mission of Automate irrigation system. This project is To develop the irrigation system to save water to the next generation. Water is an important source for life and now it’s becoming polluted and Unavailable. This means in few years’ water could be a difficult thing to get. Is south Asian countries irrigations is an open irrigation, so the Percentage of water wastage is high. Our aim is to reduce the wastage of water in irrigation. We introduce weather forecast for the irrigation. If It could be rain today so we don’t need to irrigate. We also introduce moisture sensors to find out water level on the land so it will tell you if the Plant need water or it will warn you there are too much water.

What are the requirements.

This project is mainly base on Telco platform. So the data communication is based on SMS. So the user and the device have to connect with a Telco platform. Device should have a sim-card. Before inserting the sim card you have to configure it as a device. To that please see configure the device. After configure it as a device you can put it into the device. That’s all for the configuration of the device. Then in your mobile which you going to use as a controller call *771*100# after flowing the instructions you will get the conformation message that your sim was set as controller. Then it’s over. Now you can control you home garden or your field irrigation system by just call *771*100# in your mobile phone.

Configuration

The application will be launched in live 28th Wednesday night

User to Server: all sms should be sent to 77100

Subscribe: REG da
UnSubscribe: UNREG da

configure(USSD) : *771*100#

on: da on
off: da off

if more than one devices configured

on: da on devicename
off: da off devicename

schedule: da 1110101:4.00-6.00:12.30-1.00:17.00-19.45 [devicename]
We recommend you to schedule using web interface :)
The message schedules per week first 7 bits says from Monday to Sunday which days the schedule should work and not by indicating 0 and 1. after that : separated on-off times in 24Hrs, device name is optional if user is controlling more than one device

modes of operation

1. Default mode: da mod d
Device just follow the schedule and commands
2. Alert mode: da mod a
Device just follow the schedule and commands and sends alerts+suggestions about the environment on critical situations.
3. Intelligent mode: da mod i
Device uses sensors and decide how to water the plant efficiently, only ask the user if can’t make decision.

Reporting
1. Default: da rep all
Send status sms whenevers a command/intelligent action executed.
2. Summary: da rep day send a summary sms per day da rep week send a summary sms per week
3. No reports: da rep off No status sms send back.
